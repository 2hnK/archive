# 하네스 엔지니어링 구성

## 목적

이 저장소의 하네스는 에이전트가 변경을 만들고 스스로 검증할 수 있도록 하는 최소 피드백 루프다. OpenAI의 하네스 엔지니어링 글에서 강조하는 방향처럼, 거대한 지침 파일보다 읽을 수 있는 문서 지도와 기계적 검증을 우선한다.

## 현재 구성

```sh
npm run docs:validate
npm run check
npm run build
npm run verify
```

- `docs:validate`: OKF frontmatter와 로컬 문서 링크를 검증한다.
- `check`: Astro 타입/콘텐츠 검사를 실행한다.
- `build`: 정적 사이트 빌드를 실행한다.
- `verify`: 문서 검증, Astro 검사, 빌드를 순서대로 실행하는 통합 하네스다.

## 에이전트 작업 흐름

1. `AGENTS.md`의 Required Reading 순서대로 문서 지도를 확인한다.
2. 변경 대상과 가장 가까운 기존 코드/문서 패턴을 확인한다.
3. 작업 단위를 작게 나눠 구현하거나 문서를 수정한다.
4. UI 변경이면 실행 중인 로컬 페이지에서 라이트/다크, 주요 반응형 폭, hover/focus와 스크롤 상태를 확인한다.
5. `npm run verify`로 문서와 빌드를 함께 검증한다.
6. 검증 결과와 남은 리스크를 커밋 메시지 또는 작업 보고에 남긴다.

## 포함하지 않는 것

현재는 정적 Astro 사이트이므로 다음 구성은 추가하지 않는다.

- 별도 관측성 스택
- 자동 브라우저 시각 회귀 테스트
- 원격 CI/CD 워크플로
- 대형 커스텀 린터 프레임워크

현재 UI 변경은 로컬 브라우저에서 수동 또는 에이전트 보조로 확인한다. UI 변경이 잦아지거나 같은 회귀를 반복하면 Playwright 기반 자동 화면 검증과 스크린샷 비교를 다음 하네스로 추가한다.

## 유지 규칙

- 새 문서를 추가하면 OKF frontmatter와 로컬 링크 검증을 통과해야 한다.
- 새 페이지나 컴포넌트가 생기면 관련 디자인 문서도 함께 갱신한다.
- 검증 스크립트는 에이전트가 바로 이해할 수 있는 한국어 오류 메시지를 출력한다.
- 하네스가 실제 개발 흐름보다 무거워지면 축소하거나 분리한다.
